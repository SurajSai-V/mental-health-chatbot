## Overview

MoodBoost is a mental health companion that listens, understands your emotional state, and responds with genuine empathy. It uses real-time sentiment analysis to detect how you're feeling and adapts its responses accordingly — whether you need to vent, get practical advice, or just talk.

This project was built end-to-end: frontend UI, Flask REST API backend, AI integration, Firebase database, and cloud deployment.

---

## Features

- **AI Chat** — Powered by LLaMA 3.3 70B via Groq API. Responds contextually and empathetically, not with generic scripted replies.
- **Sentiment Analysis** — Every message is analyzed in real time using TextBlob NLP. Sentiment score (-1 to +1) is displayed under each message.
- **Persistent Memory** — Full conversation history is stored in Firebase Realtime Database. The AI remembers context across sessions and server restarts.
- **CBT Counseling Corner** — Select your current mood and get structured Cognitive Behavioral Therapy guidance generated by AI.
- **Monthly Mood Analysis** — Visual chart of your sentiment trend over 30 days, plus an AI-generated summary of your emotional patterns.
- **Voice Input** — Speak your message using the Web Speech API. The app transcribes and sends automatically after a 2-second pause.
- **Chat History Sidebar** — All sessions saved and accessible, similar to ChatGPT's interface.

## How to Run Locally

**1. Clone the repo:**
```bash
git clone https://github.com/SurajSai-V/mental-health-chatbot.git
cd mental-health-chatbot
```

**2. Create virtual environment:**
```bash
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Mac/Linux
```

**3. Install dependencies:**
```bash
pip install -r requirements.txt
python -m textblob.download_corpora
```

**4. Set up environment variables:**

Create a `.env` file:
```
GROQ_API_KEY=your_groq_api_key
FIREBASE_DATABASE_URL=your_firebase_database_url
```

Add your `serviceAccountKey.json` from Firebase to the project root.

**5. Run:**
```bash
python app.py
```

Open `http://127.0.0.1:5000`

## Deployment

The app is deployed on **Render** with the following configuration:

- **Runtime:** Python 3
- **Start Command:** `gunicorn app:app`
- **Environment Variables:** `GROQ_API_KEY`, `FIREBASE_DATABASE_URL`, `FIREBASE_CREDENTIALS`
- **Auto-deploy:** Every push to `main` branch triggers a redeploy

Firebase credentials are stored as a JSON environment variable on Render — no credential files are committed to the repository.

## Key Engineering Decisions

**Why Groq instead of OpenAI?**
Groq's inference API is significantly faster (250+ tokens/second) and has a generous free tier — ideal for a real-time chat application where response speed matters.

**Why Firebase Realtime Database instead of SQL?**
Firebase's real-time sync and schemaless structure fits the dynamic nature of chat data. It also eliminates the need to manage a database server separately.

**Why Flask instead of Django?**
Flask's lightweight nature is appropriate for a single-purpose API server. Django's overhead would be unnecessary for this project scope.

**Persistent conversation history:**
Instead of storing chat history in memory (which resets on restart), every message is saved to Firebase. The AI loads the full history before each response, maintaining context across sessions and server restarts.

## Known Limitations & Future Improvements

- **No user authentication** — all users share the same Firebase instance. Next step: Firebase Auth for private per-user history.
- **Free tier sleep** — Render's free tier sleeps after 15 minutes of inactivity, causing a ~30 second cold start. Fix: upgrade to paid tier or use UptimeRobot to keep it awake.
- **Sentiment analysis accuracy** — TextBlob uses rule-based analysis and can misclassify sarcasm or complex emotions. A transformer-based model would improve accuracy.
